
1. make sure you have jdk if not install jdk first

2. Get avro-tools jar files from apache
	
3. Create executable avro-tools script and add following to it. Where you create doesn't matter may be put it in /usr/local/bin

	JAVACMD=/usr/bin/java [path to your java]
	${JAVACMD} -jar [path to avro files jar]/avro-tools-1.8.2.jar tojson $1 | tail -n $2

	echo 'export PATH=$PATH:[Path to avro-tools]' >> ~/.bashrc
	source ~/.bashrc

	Also add export PATH=$PATH to your ~/.profile


5. follow below step to install ender

	sudo apt-get install git unzip
	sudo mkdir -p /opt/dotomi/conf /opt/bin /opt/logs
	sudo chown <username>:<username> -R /opt/dotomi /opt/bin /opt/logs
	echo SYSTEM_ENVIRONMENT=PERSONAL >> /opt/dotomi/conf/master.conf

	git clone http://git.dotomi.com/scm/CM/dotomi-installer.git ~/tmp/dotomi-installer
	~/tmp/dotomi-installer/src/main/scripts/dtm-get -c installer -v RELEASE
	rm -rf ~/tmp

	echo 'export PATH=$PATH:/opt/bin' >> ~/.bashrc
	source ~/.bashrc

	Configure git as described in Git Config
	Create file ~/.code-grab.rc
	SRC=/home/<username>/Development/git
	USERNAME="<username>"
	USEREMAIL="<email>"

	dtm-get -c ender -v [ender master version : current on with this changes is 201705.MASTER.14-SNAPSHOT ] -f

6. Run ender 
	
	ender rules -rt [rule type] -p [Path to log] -u TA [All these are minimum required]

	ender rules -h -- prints available options 

	Note: Make sure no duplicates avro log for same log type. If there are multiple logs of the same log type with different version provide version as -v 23 as part of ender commnad

	If it throws memroy exception because log size is too large then you need to increase the heap size for ender application ( follow below instructions).

	 a. vim /opt/dotomi/[Ender version currently running]/bin/ctl-common
	 b. add -Xmx6g to RUN_OPTS="-server $EHCACHE_OPTS $CONFIG_OPTS $LOGBACK_OPTS $SERVER_JAVA_OPTIONS"
   		RUN_OPTS="-Xmx6g -server $EHCACHE_OPTS $CONFIG_OPTS $LOGBACK_OPTS $SERVER_JAVA_OPTIONS"
